```python
# ===== KEY PATTERN: Multi-Part Response =====
# Extract the answer text and follow-up data
answer_text = item.get('answer_text', '')
follow_up_data = item.get('follow_up_data', {})

# Create the multi-part response
parts = [
    # Part 1: Text answer for human consumption
    Part(root=TextPart(text=answer_text)),
    
    # Part 2: Structured data for follow-ups and metadata
    Part(root=DataPart(data=follow_up_data)),
]

# Send the multi-part artifact
await updater.add_artifact(parts, name='qa_response')
await updater.complete()
```

================================================================================
                           HOW IT WORKS
================================================================================

1. USER SENDS: "What is quantum computing?"

2. AGENT PROCESSES:
   - System prompt instructs LLM to generate answer + follow-ups
   - LangGraph returns structured response (Pydantic model)
   - Agent extracts: answer_text and follow_up_data

3. EXECUTOR CREATES MULTI-PART:
   - Part 1 (TextPart): answer_text → Human readable
   - Part 2 (DataPart): follow_up_data → Machine readable

4. CLIENT RECEIVES:
   - Single artifact with 2 parts
   - Can display text naturally
   - Can use data for interactive UI

================================================================================
                          CLIENT USAGE EXAMPLE
================================================================================

For each part in artifact.parts:
  
  IF part.kind == 'text':
    → Display as main answer in UI
    → Render as markdown/HTML
    → Show to user naturally
  
  ELIF part.kind == 'data':
    → Parse JSON structure
    → Render follow-ups as clickable buttons
    → Show confidence meter
    → Display topic tags
    → Use for analytics/processing
